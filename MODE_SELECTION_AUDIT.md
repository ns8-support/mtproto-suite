# Аудит выбора режима установки — install.sh

Дата проверки: 2026-07-08  
Статус: ✅ Исправлено и улучшено

---

## 📋 Исходная проблема

Пользователь сообщил: "Ранее ставилась просто нода без выбора."

## 🔍 Результаты анализа

### Найденные проблемы:

1. **`install-node.sh` автоматически устанавливает `--mode=node`**
   - Это алиас для `install.sh --mode=node`
   - При запуске `bash install-node.sh` интерактивный выбор НЕ показывается
   - Это задуманное поведение (backward compatibility), но было неочевидно

2. **Отсутствие предупреждения в `install-node.sh`**
   - Пользователь мог не понимать, что запускает установку только ноды

3. **Отсутствие проверки stdin в интерактивном режиме**
   - Если stdin не является терминалом (pipe, redirect), `read` мог зависнуть
   - Сценарий: `wget -qO- URL | bash install.sh` (неправильное использование)

4. **Документация содержит команды с `--mode=node`**
   - Пользователи копировали команды целиком, не убирая флаг

### Что работало корректно:

✅ Интерактивный выбор (1/2/3 или panel/node/both)  
✅ Парсинг CLI аргументов `--mode=panel|node|both`  
✅ Режим `-y` без `--mode` корректно завершается с ошибкой  
✅ Валидация режима перед запуском установки

## 🔧 Применённые исправления

### 1. Улучшен `install-node.sh`

**Файл:** `install-node.sh`

**Было:**
```bash
#!/bin/bash
# Backward-compat алиас для install.sh --mode=node.
exec "$(dirname "$0")/install.sh" --mode=node "$@"
```

**Стало:**
```bash
#!/bin/bash
#
# Backward-compat алиас для install.sh --mode=node.
#
# ВНИМАНИЕ: Этот скрипт автоматически устанавливает режим "node" (сервис-нода)
# без интерактивного выбора...

echo "=========================================="
echo "  Установка сервис-ноды MTProto Suite"
echo "=========================================="
echo ""
echo "Этот скрипт установит ТОЛЬКО сервис-ноду (proxy runtime)."
echo "Для выбора типа установки (панель/нода/оба) используйте:"
echo "  bash $(dirname "$0")/install.sh"
echo ""
echo "Продолжаем установку ноды через 3 секунды..."
echo "(Нажмите Ctrl+C для отмены)"
sleep 3

exec "$(dirname "$0")/install.sh" --mode=node "$@"
```

**Влияние:** Пользователь видит предупреждение и может отменить установку через Ctrl+C.

### 2. Добавлена проверка stdin в `install.sh`

**Файл:** `install.sh` → секция выбора режима

**Добавлено:**
```bash
if [ -z "$MODE" ]; then
  if [ "$AUTO_YES" != true ]; then
    # Проверка: stdin должен быть терминалом для интерактивного выбора
    if [ ! -t 0 ]; then
      echo -e "${RED}Ошибка: интерактивный выбор невозможен (stdin не является терминалом).${NC}"
      echo ""
      echo "Укажите режим явно через аргумент --mode:"
      echo "  bash install.sh --mode=panel    # Только панель"
      echo "  bash install.sh --mode=node     # Только нода"
      echo "  bash install.sh --mode=both     # Оба компонента"
      echo ""
      echo "Или запустите скрипт напрямую (не через pipe):"
      echo "  bash install.sh"
      exit 1
    fi
    # ... интерактивный выбор ...
  fi
fi
```

**Влияние:** Скрипт корректно завершается с понятным сообщением вместо зависания.

### 3. Добавлена защита от зависания в `ask()` и `ask_password()`

**Файл:** `install.sh` → функции `ask()` и `ask_password()`

**Добавлено в `ask()`:**
```bash
# Защита от зависания если stdin не терминал
if [ ! -t 0 ]; then
  if [ -n "$default" ]; then
    echo "$prompt [$default]: $default (auto — stdin не терминал)"
    REPLY="$default"
    break
  else
    echo -e "${RED}Ошибка: требуется ввод, но stdin не является терминалом.${NC}"
    echo "Используйте -y для автоматического режима или запустите интерактивно."
    exit 1
  fi
fi
```

**Добавлено в `ask_password()`:**
```bash
# Защита от зависания если stdin не терминал
if [ ! -t 0 ]; then
  PASSWORD=$(openssl rand -base64 16)
  echo "$prompt: [auto-generated — stdin не терминал]"
  return
fi
```

**Влияние:** Если скрипт запущен с `--mode=panel` (или другим) без `-y` и stdin не терминал, используются дефолты вместо зависания.

## 💡 Рекомендации для пользователей

```bash
bash install.sh
```


❌ Копирование команд из документации без удаления `--mode=node` — если нужен интерактивный выбор
